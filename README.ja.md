# Vibe Coding Coach

[English](README.md) | [中文](README.zh.md) | [日本語](README.ja.md)

> 「AIに祈りながらコードを書く」から「AIを体系的に指揮してプロダクトを作る」へ

---

## これは何？

AI コーディングアシスタント（Claude Code、Cursor、Windsurf など）用の **Skill プラグイン**。

「XX を作りたい」と言っても、AI はすぐにコードを書き始めません。
経験豊富な技術共同創業者のように、まず要件を明確にし、
6 つの仕様書を生成してから、ステップごとに構築します。

**核心価値：AI コーディングを「ランダムウォーク」から「設計図のある施工」へ。**

---

## なぜ必要なのか？

### このSkillなし

```
あなた：タスク管理アプリを作って
AI：わかりました！[500行のコードを生成]
あなた：違う、スマホ向けにして
AI：了解！[500行を書き直し]
あなた：やっぱりWeChat Mini Programで
AI：もちろん！[また書き直し]
...3日後：大量のコード、でもプロダクトなし
```

### このSkillあり

```
あなた：タスク管理アプリを作って
AI：まずいくつか確認させてください——
   誰が使う？スマホかPC？ログイン必要？
   データはどこに？参考にしたいアプリは？
   [10分のヒアリング後]
   6つの仕様書を生成しました。ステップ1から始めましょう。
...3日後：リリースできるプロダクト
```

---

## 誰のためのSkill？

### 🌱 ノーコードクリエイター
アイデアはあるがプログラミングは苦手なプロダクトマネージャー、デザイナー、起業家。

**このSkillが「監督のハンドブック」を提供します。** コードを書く必要はありません。何を作るかを知っていればいい。

### ⚡ ソロビルダー / 一人会社
プロダクト、デザイン、開発、運営を一人でこなす。時間が最も希少なリソース。

**AI協業効率を3〜5倍に：**
- 「AIが意図を推測する」時間のムダをなくす
- 「AIが導入したバグを修正する」時間のムダをなくす
- 15〜20分ごとにチェックポイント、いつでも中断・再開可能

### 👨‍💻 独立開発者 / フルスタックエンジニア
すでにAIコーディングツールを使っているが：
- 長い会話の後、AIが目標を「忘れる」
- 同じバグを3回修正しても直らない
- プロジェクトが大きくなるとAIが矛盾した決定をする

**このSkillがAIにプロジェクト記憶システムを与えます** — CLAUDE.md + progress.txt + lessons.md。

### 🏢 AI Nativeチーム
AIで開発しているが、チームコラボレーションが混乱している：
- 全員のAIワークフローが異なる
- 新メンバーがプロジェクトの「AIルール」を知らない

**チーム共有のAIコンテキスト標準を提供します。**

## コア機能

### 1. 徹底的なヒアリング
コードを書く前に、選択式＋自由回答で全ての仮定を排除。
10の余分な質問をする方が、1つの間違った仮定を持ち込むよりいい。

### 2. 6つの仕様書を自動生成

| ドキュメント | 目的 |
|------------|------|
| `PRD.md` | 機能、ユーザーストーリー、非目標 |
| `APP_FLOW.md` | 全ページ、ナビゲーション、決定ポイント |
| `TECH_STACK.md` | 全パッケージを正確なバージョンで固定 |
| `FRONTEND_GUIDELINES.md` | カラーパレット、スペーシング、コンポーネント仕様 |
| `BACKEND_STRUCTURE.md` | DBスキーマ、APIエンドポイント |
| `IMPLEMENTATION_PLAN.md` | 番号付きステップバイステップ構築シーケンス |

### 3. セッション記憶システム
- `CLAUDE.md` — AIの「プロジェクト憲法」、毎セッション自動読み込み
- `progress.txt` — 現在の状態スナップショット
- `lessons.md` — エラーログ、AIが同じミスを繰り返さない

### 4. 構造化デバッグ
詰まった？「続けて試す」ではなく：情報収集 → 仮説生成 → 検証 → **三振アウト：削除して再考**。

## 思想的背景

| ソース | 貢献した思想 |
|--------|------------|
| **Andrej Karpathy** | Vibe Codingの概念：自然言語でAIを指揮 |
| **Andrew Ng（吴恩达）** | 三振アウト原則：3回試してダメなら削除して再考（2025年インタビュー） |
| **Harper Reed** | ドキュメント駆動AIワークフロー：仕様書を先に書く |
| **Cursor/Windsurf コミュニティ** | CLAUDE.md / .cursorrules をAIコンテキストアンカーとして活用 |

## インストール

### 方法1：AIに頼む（推奨）

Claude Codeで：
```
このskillをインストールして：https://github.com/aituntian/-vibe-coding-coach
```

### 方法2：手動

```bash
git clone https://github.com/aituntian/-vibe-coding-coach \
  ~/.claude/skills/vibe-coding-coach
```
