# 审问指南

Phase 1 Step 1 的详细问题清单和选择题格式。

---

## 审问轮次（最少 3 轮，按需追加）

```
第一轮（产品定义）：
1. 这个产品是给谁用的？
2. 用户的核心行动是什么？（一句话）
3. MVP 范围是什么？哪些功能明确不做？

第二轮（数据与流程）：
4. 需要保存什么数据？数据之间有什么关系？
5. 需要展示什么数据？在哪些页面？
6. 错误时发生什么？成功时发生什么？

第三轮（技术约束）：
7. 需要用户登录吗？需要数据库吗？
8. 需要在手机上工作吗？
9. 技术栈有偏好吗？（如无偏好，根据场景推荐）
10. 有喜欢的 App/网站可以作为 UI 参考吗？

第 N 轮（追问轮 — 按需无限追加）：
- 针对前几轮回答中发现的模糊点继续深挖
- 每个模糊点用选择题呈现，附带开放选项
- 直到你能自信地说"我对这个项目没有任何假设了"才停止
```

---

## 选择题示例格式

```
关于用户登录，你倾向于哪种方案？
A. 邮箱+密码注册登录
B. 第三方登录（微信/Google/GitHub）
C. 不需要登录，匿名使用
D. 其他想法？请描述 →
```

对不清晰的每个点，给出 2-4 个选项 + 开放选项，降低用户回答成本。
