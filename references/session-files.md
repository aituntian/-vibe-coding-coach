# 会话持久文件模板与最佳实践

AI 在会话间没有记忆。这三个文件是跨会话的外部记忆，是整个体系能自运转的关键。

---

## CLAUDE.md 模板

```markdown
# 项目规则 — [项目名称]

## 项目概述
[一句话描述项目]

## 技术栈
[框架] [版本], [语言] [版本], [CSS方案], [数据库]

## 目录结构
[简要说明关键目录的用途]

## 编码规范
- [命名约定]
- [组件模式，如：使用函数组件 + hooks]
- [样式方案，如：只用 Tailwind，不用内联样式]
- [语言偏好，如：Always reply in Chinese]

## 设计令牌
- 主色：[hex]
- 背景：[hex]
- 间距基础单位：[值]

## 规范文档引用
参考以下文档获取详细信息：
- PRD.md — 产品需求和功能范围
- APP_FLOW.md — 用户流程和页面导航
- TECH_STACK.md — 锁定的依赖和版本
- FRONTEND_GUIDELINES.md — 设计系统和组件规范
- BACKEND_STRUCTURE.md — 数据库和 API 结构
- IMPLEMENTATION_PLAN.md — 逐步构建序列

## 自运转规则（关键）
- 每次会话开始：先读取 progress.txt 和 lessons.md，了解当前状态
- 每完成一个功能：提醒用户更新 progress.txt
- 每次被用户纠正：将纠正内容追加到 lessons.md，避免重复犯错
- 只构建 IMPLEMENTATION_PLAN.md 中的当前步骤，不跳步，不自由发挥
- 不引入 TECH_STACK.md 之外的依赖，除非用户明确同意
- 遇到不确定的设计决策时，先问用户，不要自行假设
- 每个新组件必须参考 FRONTEND_GUIDELINES.md 的设计令牌
- 代码中不硬编码颜色值，使用设计令牌

## 禁止事项
- 不要在前端代码中暴露 API 密钥
- 不要使用 TECH_STACK.md 之外的包
- 不要跳过 IMPLEMENTATION_PLAN.md 的步骤
- 不要在没有用户确认的情况下删除已有功能
- 不要生成超过一个功能的代码（一次一件事）
```

## progress.txt 模板

```
# 项目进度 — [项目名称]
# 最后更新：[日期]

## 已完成
- [功能描述]（[日期]）
- [功能描述]（[日期]）

## 进行中
- [功能描述] — [当前状态/卡在哪里]

## 接下来
- [下一个要做的功能，对应 IMPLEMENTATION_PLAN.md 的步骤编号]

## 已知 Bug
- [Bug 描述] — [在哪个页面/组件]

## 注意事项
- [任何后续会话需要知道的上下文]
```

更新原则：
- 每完成一个功能就更新，不要攒着
- 写清楚「做了什么」而不只是「做了」
- Bug 要写清楚复现路径
- 「注意事项」放那些容易忘记但很重要的上下文

## lessons.md 模板

```markdown
# 项目教训 — [项目名称]

## 编码教训
- [日期] [问题描述] → [解决方案] → [规则：以后怎么避免]

## 架构教训
- [日期] [问题描述] → [解决方案] → [规则]

## 工具教训
- [日期] [问题描述] → [解决方案] → [规则]

## 提示词教训
- [日期] [什么提示词效果不好] → [改成什么效果好了]
```

使用方式：
- 每次 AI 犯错被纠正后，追加一条记录
- 每次调试花了很长时间后，记录根因和解法
- 每次发现好用的提示词模式，记录下来
- CLAUDE.md 中的自运转规则会让 AI 每次会话开始时读取这个文件

---

## 自运转机制说明

这三个文件构成一个自我改进循环：

```
CLAUDE.md（规则）
    ↓ 每次会话自动读取
    ↓ 指示 AI 读取 ↓
progress.txt（状态）← AI 知道从哪里继续
lessons.md（教训）← AI 知道不要重复哪些错误
    ↓
AI 按规则工作
    ↓
完成功能 → 更新 progress.txt
被纠正 → 更新 lessons.md
发现新规则 → 更新 CLAUDE.md
```

这就是 skill 退场后体系继续运转的原理：skill 是「装修队」，来一次把房子装好；这三个文件是「房子的结构」，住在里面自然就按规矩来。
